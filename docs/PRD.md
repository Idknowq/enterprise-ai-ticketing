## 适用范围

- 定义企业级 AI 工单编排系统 MVP 的产品目标、用户角色、核心场景和验收标准。
- 指导后续迭代判断某项能力是否属于产品范围。
- 为设计、开发、测试和演示提供一致口径。

## 非目标

- 不维护接口字段、表结构和状态机细节；这些内容分别见 [API Contracts](API_CONTRACTS.md) 和 [Data Model](DATA_MODEL.md)。
- 不维护运行命令和环境变量；见 [Operations](OPERATIONS.md)。
- 不定义最终商业化完整产品，只定义当前 MVP 和可预期迭代方向。

## 背景

企业内部支持系统常见问题包括：工单分类依赖人工、知识分散、简单问题无法自动闭环、复杂请求缺少审批和审计、AI 问答难以纳入真实工作流。本项目目标不是聊天机器人，而是让 AI 参与工单流转、知识检索、流程判断、审批挂起和回复生成。

| 可追踪可信 | Product Goal | 是 | 无 | 全程记录状态、事件、引用、AI run 和审批 | 影响数据模型和安全 |

## 非目标

| 名称 | 类型 | 是否必填 | 默认值 | 说明 | 变更影响 |
| --- | --- | --- | --- | --- | --- |
| 多租户 | Non-goal | 否 | 不支持 | MVP 不实现租户隔离 | 后续加入需重构权限和数据模型 |
| 多渠道接入 | Non-goal | 否 | 不支持 | 不接入邮件、Slack、飞书等 | 后续加入需扩展入口契约 |
| 工作流设计器 | Non-goal | 否 | 不支持 | 只做固定审批模板 | 后续加入需新增产品和架构设计 |
| 自动执行内部系统操作 | Non-goal | 否 | 不支持 | AI 只生成建议和触发审批，不直接操作生产系统 | 影响安全策略 |
| 完整 BI | Non-goal | 否 | 不支持 | 只做基础可观测性和 dashboard | 影响 observability 范围 |

## 验收标准

| 名称 | 类型 | 是否必填 | 默认值 | 说明 | 变更影响 |
| --- | --- | --- | --- | --- | --- |
| 端到端工单闭环 | Acceptance | 是 | 无 | 管理员或员工可创建工单、查看详情、状态变化和时间线 | 影响 E2E |
| AI 分析可追踪 | Acceptance | 是 | 无 | 工单详情能看到 AI run、建议、引用和人工复核原因 | 影响 AI 和前端 |
| 审批闭环 | Acceptance | 是 | 无 | 需要审批的工单能进入待审批，审批通过或驳回后更新状态 | 影响 workflow |
| 权限隔离 | Acceptance | 是 | 无 | 普通员工不能访问无权限工单，审批人只看待办或相关审批 | 影响安全测试 |
| 知识引用 | Acceptance | 是 | 无 | 检索结果返回来源片段，不越权返回受限文档 | 影响知识模块 |
| 质量门禁 | Acceptance | 是 | 无 | `mvn test`、前端 lint/typecheck/test/build、E2E 和 OpenAPI smoke 按需通过 | 影响交付 |

## 维护规则

- 新增或改变用户场景、功能范围、验收口径时，先更新本文件。
- 需求落到接口、数据或模块后，必须同步更新相关事实源文档。
- P0/P1 需求没有验收标准不得进入开发。
